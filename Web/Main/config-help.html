Описание параметров и настроек теплового насоса


Расположение файлов веб-сервера на SPI Flash - в НК Web интерфейс управления ТН, можно расположить в двух местах:
- на SD карте, там же хранится история работы ТН, которая пишется раз в минуту (чисто теоретически это менее надежное хранение);
- в микросхеме flash SPI, если она установлена.
Выбор данного чекбокса говорит, что НК пытаться загружать интерфейс с чипа памяти SPI, в противном случае будет производиться загрузка с SD карты. Надо понимать, что эта опция задает только приоритет загрузки, т.е. если на чипе памяти SPI файлы не найдены, будет произведена попытка загрузки с SD карты.
Загрузить файлы Web интрефейса можно на странице СЕРВИС->ФАЙЛЫ. На SD карту можно просто скопировать необходимы файлы через кард ридер, карта должна быть отформатирована в fat32 или fat16, желательно использовать карту маленького размера (1-4 Гб), и при форматировании указать максимальный размер кластера, для оптимизации скорости работы в НК.
Сохранение состояния ТН в ЕЕПРОМ, для восстановления его после сброса - Эта опция сохраняет чекбокс включения ТН (тумблер на панели управления сверху) во флеш памяти. При сбросе НК, это состояние будет восстановлено. Если этот чекбокс не выбран, ТН работает, и отключили электричество, то после его включения ТН останется в выключенном состоянии. Обычно эту опцию надо включить, для увеличения отказоустойчивости НК.
Число повторных попыток пуска при ошибке - Сколько раз, ТН будет пытаться перезапустится, при ошибке или сбросе. Обычно устанавливется значение 1-5 раз, если 0 то НК не будет пытаться перезапуститься.
В файле CONFIG.h #define NOT_RESTART_ON_CRITICAL_ERRORS в ней, можно перечислить критические ошибки, при их появлении ТН остановится и рестарта не будет.
Интервал накопления точек графиков в памяти - Для оперативного контроля и настройки НК сохраняет данные о работе ТН в оперативной памяти и их можно посмотреть в виде графиков на странице ОТЛАДКА ТН -> ГРАФИКИ. Данная настройка устанавливает интервал времени между двумя соседними сохранениями (интервал между двумя точками графика). После сброса или пуска ТН графики стираются из оперативной памяти.
Запись графиков в память, только во время работы компрессора - При включенной опции, графики в оперативной памяти пишутся только когда работет компрессор, что позволяет уместить больше информации, но теряют информацию в паузах и временная шкала на графике будет иметь разрывы.
Запись истории на SD карту -данная опция включает запись истории (телеметрии ТН) с интервалом 1 минута на SD карту НК. При этом история пишется постоянно, файлы формируются по годам. Посмотреть историю в графическом виде можно на странице СТАТИСТИКА ->ДЕТАЛЬНО.

Настройки контура отопления - блок настроек описывающих дополнительные настройки контура отопления
Использование дополнительного ТЭНа при нагреве отопления - эта опция определяет использование дополнительного электро котла (стандартное имя RHEAT) в контуре отопления. Возможно три варианта выбора:
none - не использовать электро котел;
reserve - резервирование отопления ТН, если температура в помещение опускается ниже Температуры для управления дополнительным ТЭНом, включается электро котел;
bivalent - электро котел включается когда уличная температура опускается ниже Температуры для управления дополнительным ТЭНом. Эта опция сделана для воздушных ТН, при низких уличных температурах производится дополнительный нагрев электрокотлом.
Температуры для управления дополнительным ТЭНом - температура при котороий включается электро котел, имеет различный функционал в зависимости от режима работы (см предыдущий пункт)
Работа насоса отопления, когда ТН в паузе - при наличии теплоаккумулятора в системе отопления эта опция задает время работы насоса отопления в паузе (ТН не работает), при которой производится отбор тепла от теплоаккумулятора в дом. Если установлен 0 - насос СО не будет включатся.
Период останова насоса отопления в паузе - при наличии теплоаккумулятора в системе отопления эта опция задает время паузы насоса отопления в паузе работы ТН Если установлен 0 - насос СО будет работать не прерывно. Задавая эти два времени можно настроить требуемый отбор тепла от теплоаккумялора в дом.

Солнечный Коллектор - опции для настройки солнечного коллектора (СК), если он присутствует в системе и есть поддержка в прошивке НК (#define USE_SUN_COLLECTOR файл config.h). СК может использоваться двумя способами: 1. При работе на отопление добавлять тепло в гео контур; 2. Для регенерации (восстановления) геоконтура - 'закачка' тепла в гео контур для отогрева грунта вокруг него, что приведет к более эффективной работе ТН.
Температура воздуха (TSUN) выше которой открывается СК - температура при которой начинает работать СК.
Температура воздуха (TSUN) ниже которой закрывается СК - температура при которой отключается работа СК.
Минимальное время работы, после которого будут проверятся температуры - время контроля температуры СК для его гарантированного открытия (начала работы)
Минимальное время паузы после останова СК - минимальное время пауза, для ограничения частоты включения СК
Использовать СК для регенерации геоконтура в простое - включение опции регенерации гео контура (сброса тепла из СК в геоконтур).
Температура воздуха (TSUN) начала регенерации геоконтура с помощью СК (+Δ) - температура начала регенерации гео контура.
Температура (TSUNOUTG) завершения регенерации геоконтура с помощью СК - температура прекращения регенерации гео контура
Разница (Δ) температур: TSUN и на выходе в геоконтур для включения СК - дополнительное условие на включение регенерации гео контура.
Разница (Δ) температур на выходе из СК и на выходе в геоконтур для выключения СК- дополнительное условие на выключение регенерации гео контура.

Времена и задержки - блок опция описывающая временные параметры.
Минимальное время простоя компрессора перед включением - Для обеспечения разгруженного пуска компрессора, после его остановки. Задается время, за которое выравниваются давления высокое с низким, также это минимизирует число пусков компрессора и сберегает его ресурс. Обычно устанавливается 2-10 минут. Обмотки статора компрессора, особенно не инверторного, при пуске значительно разогреваются высокими токами.
Минимальное время работы компрессора - необходимо, чтобы гарантированно охладить двигатель парами фреона. Чем мощнее компрессор, тем выше пусковой ток и больше время минимальной работы. Также это минимизирует число пусков компресора и сберагает его ресурс. Обычно устанавливается 5-15 минут. Очень актуально для старт-стопного ТН, что бы он не 'стартопил'.
Задержка включения компрессора после включения насосов - Необходима, для обеспечения необходимого протока в испарителе и конденсаторе ТН (инерционность системы) и обеспечения валидных данных с датчиков протока (проток, должен успеть считаться) и температуры. Иначе останов по минимальному протоку после пуска и не точная регулировка ЭТРВ в режиме Стартовая позиция ЭРВ определяется по температуре подачи.
Задержка выключения насоса СО, после выключения компрессора - Инверторные компрессора, после команды стоп не моментально, полностью останавливаются. В зависимости от настроек времени выбега привода, задается задержка + 15...30 сек.
Задержка включения ТН после внезапного сброса контроллера - данная задержка необходимо что бы ТН после сброса (например отключили электричество) пришел в 'равновесное' состояние, выравнил давления и плавно стартовал. Обычное значение 5-15 минут.
Задержка перед повторным включением ТН при ошибке (попытке пуска) - Учитывает, индивидуальные особенности ТН. Время всех процессов которые должны завершится на момент повторного пуска ТН. Как допустим достижение крайних положений 3х ходового клапана ГВС итд... Обычное значение 5-15 минут.
Чтобы при переключении режимов: отопление - разморозка, отопление - охлаждение и на оборот, избежать гидравлического удара в фреоновом контуре.
Задержка между переключением реверсивного клапана и включением компрессора - За это время, должны выровняться давления высокое с низким. Есть вероятность, что в режиме ГВС теплоноситель прокачиваемый в конденсаторе, имеет температуру выше защит температур СО.
Пауза после переключения ГВС - В это время контроллер будет ожидать, падения температуры теплоносителя на выходе конденсатора после режима ГВС (подбирается экспериментально).
Пример: Клапан приоритета ГВС имеет время переключения 5 сек + время насосу прокачать конденсатор + инерционность датчика температуры ТCONOUTG закрепленного в каждом ТН индивидуально. Сумма времен этих нюансов, будет паузой.
Время на сколько блокируются защиты при переходе с ГВС на отопление - время на которое блокируются защиты отопления после переключения с ГВС, что бы температуры вышли на приемлемый уровень для отопления, иначе они будут срабатывать и будет возникать ошибка

Дополнительное оборудование
Работа ТН от резервного источника питания (генератор) - включение данной опции приводит к ограничению потребляемой мощности ТН. Это необходимо при работе от резервного источника, например от генератора. Генератор обычно имеет мощность меньше чем максимальное потребление ТН, по этому что бы генератор не выключался по превышению нагрузки мощность потребления ТН надо ограничить величиной меньшей максимальной мощности генератора.
Если на НК не заведен сигнал перехода на работу от генератора (вход SGENERATOR) то необходимо в ручную установить эту опцию при работе от генератора. Для удобства данный чекбокс продублирован на странице СОСТОЯНИЕ (генератор) и на стартовом экране дисплея Nextion (см. рисунок - кнопка с изображением генератора)
В НК предусмотрена возможность информирования по HTTP протоколу других систем (умный дом) о работе от резервного источника питания, например, системы водоснабжения, чтобы она одновременно насосы не включала.
Включается через файл config.h (формат: сервер/запрос), например:
#define HTTP_LowConsumeRequest "192.168.0.8/&LowConsume_set="
Будет выполнен HTTP GET запрос, в конец которого будет подставлено '1', если питание от резерва или '0', если нет.
Автоматическое определение работы от резервного источника - эта опция включает автоматическое ограничение мощности. Для этого надо понимать что к НК (вход SGENERATOR) должен быть подключен сигнал перехода на работу от генератора, без этого сигнала автоматический переход не возможен.
Максимальная мощность потребления ТН от резервного источника питания величина ограничения мощности при работе от резервного источника.
Логировать не критичные ошибки электро-счетчика SDM - включение данной опции приведет к записи ВСЕХ ошибок электро счетчика в журнал, в том числе ошибок чтения/записи, это увеличивает и 'засоряет' журнал но может потребоваться для отладки. Обычно данная опция выключена.
Логировать обмен между беспроводными датчиками - включение данной опции приведет к записи обмена с беспроводными датчиками в журнал, это увеличивает журнал но может потребоваться для отладки. Обычно данная опция выключена.
Генерация звукового сигнала - включение данной опции разрешает генерацию звукового сигнала при ошибке (прирывистый писк). Обычно эта опция включена.
Использование дисплея Nextion - включить использование внешнего дисплея (прошивка должна быть собрана с поддержкой дисплея #define NEXTION в файле config.h). Обычно эта опция включена.
Включать дисплей при пуске компрессора - обычно дисплей засыпает через определенный интервал времени. Включение данной опции будит дисплей при включении компрессора. Можно по включению дисплея определять включение компресора.
Время засыпания дисплея Nextion - время через которое дисплей переводится в режим сна. Пробуждение дисплея происходит при нажатии на него, сбросе НК, или если включена опция Включать дисплей при пуске компрессора и компрессор включился.
Яркость дисплея Nextion в % - задается комфортная яркость дисплея.
На первой шине 1-Wire только один датчик температуры - НК поддерживает до четырех независимых шин 1-Wire. Данная опция говорит НК что на данной шине присутствует только один датчик, и можно проводить его опрос по упрощенной схеме.
На второй шине 1-Wire только один датчик температуры - Данная опция говорит НК что на данной шине присутствует только один датчик, и можно проводить его опрос по упрощенной схеме.
На третьей шине 1-Wire только один датчик температуры - Данная опция говорит НК что на данной шине присутствует только один датчик, и можно проводить его опрос по упрощенной схеме.
На четвертой шине 1-Wire только один датчик температуры - Данная опция говорит НК что на данной шине присутствует только один датчик, и можно проводить его опрос по упрощенной схеме.
