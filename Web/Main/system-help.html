Описание параметров Системы


Параметры системы
Страница предназначена для вывода различных системных переменных НК. Данная страница позволяет пользователю НК проводить мониторинг текущего состояния НК и смотреть различные служебные переменные ПО.

КНОПКА Подготовка к обновлению - при нажатии на эту кнопку происходит проверка включения компрессора ТН. Если компрессор включен то выскакивает сообщении о необходимости его выключения. Если компрессор выключен, то сохраняется статистика (подробнее см. "Статистика за день").
КНОПКА Сохранить статистику - при нажатии на эту кнопку сохраняется статистика (подробнее см. "Статистика за день").

Время с последней перезагрузки - показывает сколько времени прошло с последней перезагрузки, используется для оценки не контролируемых перезагрузок при длительной работе НК
Последний сброс - время и дата последней перезагрузки НК. Всегда сравниваем с датой (хотя бы примерной), когда Вы перегружали НК, при не совпадении ищем причину перезагрузки.
Причина последнего сброса контроллера- указывает причину (источник) последнего сброса. Может быть:
 General - Общий сброс происходит при обнаружении сброса при включении питания, асинхронного общего сброса (вывод NRSTB), обнаружения источником питания пониженного напряжения или потерю регулирования напряжения.
 Backup - Сброс резервного сохранения, когда контроллер возвращается из режима резервного копирования. Сигнал core_backup_reset утверждается контроллером питания, когда происходит резервный сброс.
 Watchdog - Сброс при срабатывание сторожевого таймера (Watchdog).
 Software - Спрос по команде программы (программный сброс), осуществляется путем записи определенных битов в регистр RSTC_CR.
 User - Пользовательский сброс проводится, когда на выводе NRST обнаруживается низкий уровень и бит URSTEN в RSTC_MR равен 1.
 Unknown - этого не должно быть, заглушка для непонятного источника сброса, проблема с кодом НК.
Для НК актуальны General (нажатие кнопки reset, подача питания, плохое питание), Watchdog (программа либо зависла либо есть ошибки в коде) и Software (программный сброс через веб морду)
Температура платы (DS3231) - микросхема часов DS3231 имеет встроенный датчик температуры для термокомпенсации, здесь выводится показание этого датчика, показывает примерную температуры платы.
Свободная память - показывает свободную память системы free RTOS. Расчет достаточно приблизительный. При величине более 10к можно увеличивать внутренние массивы типа точек на графиках и число потоков web сервера.
Источник загрузки web интерфейса - определены следующие варианты источника загрузки web морды:
 internal - загрузка одно страничной web морды из самой прошивки НК (содержится внутри кода). Это аварийный вариант когда другие источники не доступны.
 SD card - загрузка с установленной SD карты
 SPI Flash - загрузка с чипа SPI flash (если он установлен на плате), есть разные размеры 4 или 16 Мб.
 unknown - этого не должно быть, заглушка для непонятного источника, проблема с кодом НК.
Источник загрузки можно выбрать СЕРВИС->НАСТРОЙКИ->"Расположение файлов веб-сервера на SPI Flash (иначе на SD карте)"
Входное напряжение питания контроллера - если НК позволяет измерять напряжение источника питания (это опция) то выводится его напряжение.
Режим safeNetwork (адрес:192.168.0.177 шлюз:192.168.0.1, не спрашивать пароль) - показывает признак того что НК находится в режиме safeNetwork (заранее заданные сетевые настройки).
Контроль за работой драйвера ЭРВ - если НК позволяет (в качестве драйвера ЭРВ используется L9333MD) выводится состояние этого драйвера. Ошибка драйвера может быть при перегреве микросхемы и замыкания обмоток ЭРВ.
Состояние FreeRTOS при старте (task+err_code) - в НК встроен механизм для отладки, при внезапном сбросе или падении free RTOS НК делает попытку (не всегда удачную) сохранить номер задачи и причину падения. Надо изучать при внезапных сбросах и зависаниях НК
Старший байт номер задачи (номер см. на странице ОС RTOS), младший байт код ошибки (1-configASSERT fails, 2-malloc fails, 3-stack overflow, 4-hard fault, 5-bus fault, 6-usage fault, 7-crash data)
Регистры контроллера питания (SUPC) SAM3X8E [SUPC_SMMR SUPC_MR SUPC_SR] - значение регистров управления питанием контроллера (см даташит на SAM3X)
Строка статуса ТН - показывает что в настоящее время делает НК и почему.
  State: 0 - Выключен, 1 - Стартует, 2 - Останавливается, 3 - Работает, 4 - Ожидание, 5 - Ошибка.
  modWork:
  Перечисляемый тип - точка возврата из алгоритма управления (расшифровка состояния)
  Расшифровка кода (пример Bp12 Бойлер - Алгоритм ПИД - изменение частоты ПИДом)
  Код имеет три поля <источник><алгоритм><код_алгоритма>
  Источник: B-бойлер, H-отопление, C-охлаждение
  Алгоритм: h - гистерезис, p - ПИД
  Код алгоритма
  1 - выключение по подаче
  2 - включение по гистерезису
  3 - выключение по гистерезису
  4 - внутри гистерезиса (продолжение: нагрев или охлаждение)
  5 - внутри гистерезиса пауза
  6 - сброс частоты по подаче
  7 - сброс частоты по мощности
  8 - сброс частоты по температуре компрессора
  9 - сброс частоты по давлению
  10 - разгон, пид не работает
  11 - время пида не пришло
  12 - дошли до ПИДа, регулируем
  13 - включение по обратке достигнута минимальная температура обратки
  14 - работа супербойлера ПИД ГВС (заход в бойлер)
  15 - Бойлер греется от предкондесатора (заход в отопление)
  16 - сброс частоты по току инвертора
  17 - блокировка роста частоты ПИДом при подходе к уровням защиты ПОДАЧА
  18 - блокировка роста частоты ПИДом при подходе к уровням защиты МОЩНОСТЬ
  19 - блокировка роста частоты ПИДом при подходе к уровням защиты ТОК
  20 - блокировка роста частоты ПИДом при подходе к уровням защиты ТCOMP
  21 - блокировка роста частоты ПИДом при подходе к уровням защиты ДАВЛЕНИЮ
  22 - Выключение нагрева бойлера ТН для перехода в режим ДОГРЕВА его ТЭНом
  23 - Выключение режима ТН при достижении уровня защиты по подаче (достижение границы)
  24 - Выключение режима ТН при достижении уровня защиты по мощности (достижение границы)
  25 - Выключение режима ТН при достижении уровня защиты по температуре компрессора (достижение границы)
  26 - Выключение режима ТН при достижении уровня защиты по давлению (достижение границы)
  27 - Выключение режима ТН при достижении уровня защиты по току (достижение границы)
  28 - Ограничение мощности при работе от резервного источника питания (сброс частоты)
  29 - Выключение режима ТН при достижении минимальной частоты при работе от резервного источника питания.
  30 - Ограничение при работе от генератора.
fHP(0x..): 01 - BoilerTogetherHeat, 02 - SunNotInited, 04 - SunSwitching, 08 - SunReady, 10 - SunWork.

Времена
Текущее время - время установленное в НК
Время последнего включения ТН - когда последний раз был включен ТН (кнопка HP OFF/HP ON)
Время текущего состояния ТН - когда ТН перешел в состояние которое показана в "Строке статуса ТН".
Время старта компрессора - время включения компрессора ТН
Время останова компрессора - время выключения компрессора ТН
Время сохранения текущих настроек ТН - последнее время сохранения настроек в EEPROM

Счетчики ошибок различные счетчики ошибок которые позволяют проводить анализ надежности работы НК и выявлять проблемы с помехами. Счетчики сбрасываются при пропадании питания.
Счетчик текущего числа повторных попыток пуска ТН - если НК был остановлен по ошибке или пропадало питание, этот счетчик показывает с какой попытки был осуществлен успешный пуск ТН.
Счетчик "Потеря связи с W5500", повторная инициализация - как часто терялась связь (обмен) контроллера SAM3X8E с сетевым чипом W5500 и проводилась повторная инициализация. Обычно 0, при частоте больше 2-3 раз в сутки необходимо разбираться с проблемой.
Счетчик числа сбросов мютекса захвата шины SPI - показывает конфликты задач free RTOS при захвате шины SPI. Обычно 0, при частоте больше 5-10 раз в сутки необходимо разбираться с проблемой.
Счетчик числа сбросов мютекса захвата шины I2C - показывает конфликты задач free RTOS при захвате шины I2C. Обычно 0, при частоте больше 5-10 раз в сутки необходимо разбираться с проблемой.

Счетчик числа повторных соединений MQTT клиента - показывает качество связи с сервером MQTT или пропадание интернета в локальной сети.
Счетчик неудачных ping - при пинговании шлюза показывает качество связи в локальной сети (должно быть 0), при пинговании внешнего сервера показывает пропадание интернета в локальной сети и/или качество пингуемого узла.
Счетчик числа ошибок чтения электро-счетчика - показывает качество связи с электросчетчиком по rs485. Обычно такие ошибки есть, вызывает беспокойство больше 50-100 ошибок в день.
Счетчик числа ошибок чтения частотного преобразователя (RS485)- показывает качество связи с электросчетчиком по rs485. Обычно такие ошибки есть, вызывает беспокойство больше 10-30 ошибок в день.
Счетчик числа ошибок чтения датчиков температуры (DS18b20) - показывает качество связи с датчиками температуры по шине oneWire. Обычно такие ошибки есть, вызывает беспокойство больше 100 ошибок в день на один датчик. На странице КОНФИГУРАЦИЯ ТН->ДАТЧИКИ ТЕМПЕРАТУРЫ можно посмотреть статистику по каждому датчику. Обычно большое количество ошибок приходится на наиболее удаленные датчики (длинный провод). Если ошибки чтения происходят часто на одном датчике это признак или большого количества помех или то что он скоро откажет.

Глобальные счетчики (Всего за весь период) НК имеет две группы счетчиков (глобальные и сезонные) которые хранятся в энергонезависимой памяти.
Первая группа это глобальные счетчики которые ведут отсчет с момента пуска ТН в эксплуатацию. Сбросить эти счетчики возможно выполнив специальный запрос через web морду ip/RESET_CNT_ALL
Время сброса счетчиков - дата и время сброса глобальных счетчиков.
Часы работы ТН (час) - время работы ТН (включения его - кнопка на web морде "HP ON")
Часы работы компрессора ТН (час) - время работы компрессора ТН (для оценки ресурса компрессора)
Потребленная энергия ТН (кВт*ч) - сколько электроэнергии использовал ТН при работе
Выработанная энергия ТН (кВт*ч) - сколько тепла выработал ТН при работе. Поделив эту величину на предыдущий параметр получим усредненный COP ТН с момента пуска ТН в эксплуатацию.

Сезонные счетчики
Вторая группа счетчиков, сбрасываются по кнопке в конце страницы СИСТЕМА. Обычно используются в НК для подсчета данных по одному отопительному сезону, и сбрасываются в начале отопительного сезона (осень).
Время сброса сезонных счетчиков ТН- дата и время сброса сезонных счетчиков.
Часы работы ТН за сезон (час) - время работы ТН (включения его - кнопка на web морде "HP ON")
Часы работы компрессора ТН за сезон (час) - время работы компрессора ТН (для оценки ресурса компрессора)
Потребленная энергия ТН за сезон (кВт*ч)- сколько электроэнергии использовал ТН при работе за сезон
Выработанная энергия ТН за сезон (кВт*ч) - сколько тепла выработал ТН в текущем сезоне. Поделив эту величину на предыдущий параметр получим усредненный COP ТН в текущем сезоне

Статистика за день
НК непрерывно накапливает статистику своей работы. Каждый день накопленная информация сохраняется на SD карту в файл (формат csv, информация хранится по дням), т.е. в течении дня статистика накапливается в оперативной памяти контроллера, а в конце дня записывается в файл с именем stats_XXXX(XXXX - год). Файл хранит информацию одного года по дням, с нового года открывается новый файл. Файл статистики можно выгрузить со страницы СЕРВИС->ФАЙЛЫ->"Статистика работы ТН (файлы по годам)". Т.к информация о текущем дне хранится в оперативной памяти, то при сбросе она теряется. Если Вам надо сбросить НК и Вы не хотите потерять эту информацию, то перед сбросом нажмите кнопку "Сохранить статистику" на странице СЕРВИС->СИСТЕМА.
Параметры ниже отображают накопленную статистику за текущий день.
Т° улицы (Мин) - минимальная температура улицы с 0 часов по текущее время.
Т° улицы (Сред) - средняя температура улицы с 0 часов по текущее время
Т° улицы (Макс) - максимальная температура улицы с 0 часов по текущее время
Т° в доме (Сред) - средняя температура в доме с 0 часов по текущее время
Т° на входе испарителя по гликолю (Сред)(work) - средняя температура при работе которая приходит на испаритель (из грунта) с 0 часов по текущее время
Т° в бойлере ГВС (Сред) - средняя температура ГВС с 0 часов по текущее время
Выработано, кВтч(work) - сколько выработано тепла с 0 часов по текущее время
Потребление, кВтч - сколько потреблено электроэнергии с 0 часов по текущее время
Потребление, кВт (Макс) - максимальная (потребленная) мощность с 0 часов по текущее время
Полный COP (Мин)(work) - минимальный СОР с 0 часов по текущее время
Полный COP(work)
- СОР при работе ТН с 0 часов по текущее время Напряжение, V (Мин) - минимальное входное напряжение с 0 часов по текущее время.
Напряжение, V (Сред)- среднее входное напряжение с 0 часов по текущее время.
Напряжение, V (Макс)- максимальное входное напряжение с 0 часов по текущее время.
Моточасы, м - МИНУТЫ работы компрессора с 0 часов по текущее время.

КНОПКА Сброс контроллера - проведение программного сброса НК (причина сброса - Software). Если надо сохраните накопленную статистику.
КНОПКА Сброс сети - инициализация сетевого чипа w5500
КНОПКА Сброс сезонных счетчиков - обычно проводится в начале отопительного сезона, предварительно если надо сохраняются (записываются на бумажке) сезонные счетчики предыдущего сезона. Для проведения сброса в соседнее с кнопкой поле надо ввести слово RESET, и нажать кнопку сброса сезонных счетчиков.
